HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The content is between an opening tag and a closing tag. Closing tags begin with a forward slash
  • Elements inside the body element are described as a child element
  • Elements inside the body element include the p element, which represents a paragraph
  • Also, inside the body element, there are ul, ol, and li elements to represent unordered lists, ordered lists, and list items.
  • The img element has all the info about images that will be shown on the website
  • Inside the img element, there is an important atrribute called src. This attribute defines the location of the image
  • To create a line of empty space between two paragraphs, we use the "br" element.
  • To insert a hyperlink, the "a" element is used followed by the "href" attribute that points to the URL link

CSS

  • There is 3 ways to style a webpage: inline CSS, internal CSS style sheet, and external CSS style sheet
  • most common way to style a webpage is by using an external CSS style sheet
  • in the "head" element, the file's link has to be placed with the href attribute
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector

Git

  • Git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git branch: list your branches. a* will appear next to the currently active branch
  • git rm (file): delete the file from project and stage the removal for commit
  • deleting a branch: git branch -d branch-name
  • git commit: saves your changes locally. it is like setting a checkpoint in the development process
  • git push: uploads your commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • each line of JavaScript code ends with a semicolon
  • code runs from the first statement at the top of the page, down to the last statement at the bottom of the page
  • use conditional statements and loops to control the order in which code is executed
  • An array is a single variable that is used to hold a group of data
  • single array can store different types of data, such as strings, numbers
  • "For" loop follows these three statements:First statement determines the starting point for our loop. Second statement is the condition. Final statement allows the array to iterate over each item
  • A function's name is followed by parentheses
